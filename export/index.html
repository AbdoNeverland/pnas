FAQ
What is the purpose of using SMS verification for user phone numbers?

SMS verification is used to ensure that the phone number provided by the user is valid and belongs to them, adding an extra layer of security to the verification process.

How does the RESTful API for sending SMS messages work?

The RESTful API allows developers to programmatically send SMS messages to users' phone numbers for verification purposes by making HTTP requests to the API endpoints.

Is there a limit to the number of SMS messages that can be sent using the API?

Yes, there may be limits on the number of SMS messages that can be sent using the API based on the pricing plan chosen by the user. Users can check the pricing details for more information.

Can users customize the content of the SMS messages sent for verification?

Yes, users can customize the content of the SMS messages sent for verification to include specific instructions or branding elements as needed.

What happens if a user does not receive the SMS verification code?

If a user does not receive the SMS verification code, they can request a new code to be sent or choose alternative verification methods such as email or phone call verification.
